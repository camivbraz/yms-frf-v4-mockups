5.2  Logs
Notification Log Log all notifications sent to drivers across the system.

Fields: notification_id, type, driver_id, queue_id, trigger_event, timestamp, delivery_status, confirmation_status, confirmation_timestamp.

Events covered:
  • Gate Queue Control (4.3.1): gate space available (1st attempt + retries), On Hold after retry limit reached
  • End Process (4.3.2): Progress Alert, End Process, Pre-Call, Effective Call
  • Entry site (4.2.3): entry tag assigned, cascade update, early arrival blocked/released
  • Existing queue notifications: dock assignment, dock reassignment, queue On Hold, return from On Hold
Entry Tag Log Log all entry tag state changes per driver.

Fields: log_id, driver_id, queue_id, previous_tag, new_tag, trigger_event_type, operator_id, timestamp.

Events covered:
  • Initial entry tag assignment
  • Cascade reassignment (system-triggered)
  • On Hold placement (entry tag removed)
  • Return from On Hold (entry tag reassigned)
  • Manual tag change by operator
  • Entry tag cleared on queue closure
Geofence Events Log Log all geofence crossing events per driver.

Fields: event_id, driver_id, geofence_id, geofence_type, event_type (entry/exit), timestamp, queue_status_at_event.

Geofence types covered:
  • Station Geofence: entry and exit
  • Entry Site Geofence (per site): entry and exit — linked to entry tag assignment and cascade logic
  • Gate Area Geofence — Portaria (4.3.1): entry and exit — used to calculate gate occupied_meters in real time
5.3  Database Schema
New tables 1. dock_attribute_cards
dock_id, attribute_type, selected_values[ ], min_threshold, max_threshold. One row per attribute card per dock.

2. entry_site_vehicle_pools
entry_site_id, vacancies, vehicle_types[ ]. One row per pool per entry site.

3. gate_queue_config
station_id, gate_capacity_meters, confirmation_timeout_min, max_travel_time_min, retry_limit. One row per station.

4. process_notification_config
station_id, task_type, notification_type, enabled, threshold_pct (X₁ and X₂). One row per notification type per task type per station.

5. user_column_preferences
user_id, column_id, visible, display_order. One row per column per user.

6. notification_log
notification_id, type, driver_id, queue_id, trigger_event, timestamp, delivery_status, confirmation_status, confirmation_timestamp.

7. entry_tag_log
log_id, driver_id, queue_id, previous_tag, new_tag, trigger_event_type, operator_id, timestamp.

8. geofence_events_log
event_id, driver_id, geofence_id, geofence_type, event_type (entry/exit), timestamp, queue_status_at_event.
